ReadOnlyExtendedBitFieldAttribute

ReadOnlyExtendedBitFieldAttribute combines the functionality of ReadOnlyBitFieldAttribute and ExtendedBitFieldAttribute. It provides bit-packing for read-only properties while allowing values larger than the allocated bit size by redirecting to an external getter.

What is ReadOnlyExtendedBitFieldAttribute?​

‚úÖ Inherits from both ReadOnlyBitFieldAttribute and ExtendedBitFieldAttribute
‚úÖ Supports all standard bit-packing features from BitFieldAttribute
‚úÖ Automatically generates constructors for read-only properties
‚úÖ Allows properties to store values larger than allocated bits
‚úÖ Uses an external getter when values exceed the bit limit
‚úÖ Requires GetterLargeSizeValueName for overflow handling
‚úÖ Supports FieldName for referencing existing read-only fields

Example: Read-Only Bit-Packed Properties with Large Values​

public sealed partial class ReadOnlyExtendedBitFieldExample
{
    private readonly int _existingField;

    [ReadOnlyExtendedBitField(BitsCount = 5, GetterLargeSizeValueName = nameof(GetMaxValue))]
    public partial int Data1 { get; }

    [ReadOnlyExtendedBitField(BitsCount = 10, GetterLargeSizeValueName = nameof(GetMaxValue), ConstructorAccessModifier = AccessModifier.Public)]
    public partial int Data2 { get; }

    [ReadOnlyExtendedBitField(BitsCount = 6, FieldName = nameof(_existingField), GetterLargeSizeValueName = nameof(GetMaxValue), ConstructorAccessModifier = AccessModifier.Public)]
    public partial int Data3 { get; }

    public static int GetMaxValue() => 1024; // External getter for overflow values
}

How It Works​

1Ô∏è‚É£ BitsCount allocates bits for storage, but if the value exceeds the max, the getter calls GetterLargeSizeValueName.
2Ô∏è‚É£ ReadOnlyExtendedBitFieldAttribute ensures immutability by allowing only read-only properties.
3Ô∏è‚É£ Constructors are automatically generated, grouping properties with the same ConstructorAccessModifier.
4Ô∏è‚É£ FieldName allows referencing existing fields, but they must be readonly.

Common Errors​

1Ô∏è‚É£ Referencing a Non-Readonly Field​

If FieldName references a non-readonly field, an error occurs:

PRBITS013
Invalid reference to non-readonly field in 'ReadOnlyBitFieldAttribute.FieldName'.
The 'FieldName' for property {0} must reference a readonly field when using the nameof operation.

‚úÖ Fix: Ensure the field is readonly.

private readonly int _bitField; // ‚úÖ Allowed

[ReadOnlyExtendedBitField(FieldName = nameof(_bitField), BitsCount = 8, GetterLargeSizeValueName = nameof(GetMaxValue))]
public partial int Data { get; }

If a property has a setter, it must be init-only or read-only. Otherwise, an error occurs:

PRBITS014
ReadOnlyBitFieldAttribute requires property without setter or with init-only setter.
The property {0} with ReadOnlyBitFieldAttribute must either be read-only or have an init-only setter.

üö´ Invalid Code (Causes Error)

[ReadOnlyExtendedBitField(BitsCount = 6, GetterLargeSizeValueName = nameof(GetMaxValue))]

Why Use ReadOnlyExtendedBitFieldAttribute?​

‚úÖ Ensures immutability ‚Äì Values set only through constructor or external getter
‚úÖ Optimized for large values ‚Äì Handles numbers beyond allocated bits
‚úÖ Automatically generates constructors ‚Äì Groups properties with same ConstructorAccessModifier
‚úÖ Uses FieldName to pack into existing fields ‚Äì But they must be readonly
‚úÖ Defaults to private constructor unless explicitly changed

Important Notes​

‚ö† ReadOnlyExtendedBitFieldAttribute inherits from both ReadOnlyBitFieldAttribute and ExtendedBitFieldAttribute, meaning it supports all features of both.
‚ö† The generator automatically creates constructors based on ConstructorAccessModifier.
‚ö† If a property exceeds its bit size, the getter redirects to GetterLargeSizeValueName.
‚ö† By default, the constructor is private, unless ConstructorAccessModifier is specified.
